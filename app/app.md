# 产品需求文档 (PRD)：饮水记录 App “喝”

## 1. 项目概述
* **应用名称：** 喝
* **开发技术栈：** Kotlin + Jetpack Compose (Android 原生)
* **核心功能：** 设定饮水目标、交互式记录单次饮水、日历数据展示与管理、支持浅/深色模式及数据导入导出。

## 2. 核心页面与功能要求

### 2.1 主界面 (首页)
* **顶部导航与状态区：** * 左上角：日历图标（点击跳转至历史记录页）。
    * 右上角：设置图标（点击跳转至设置页）。
    * 中央顶部：线性进度条，动态展示当日已记录饮水量占“每日目标进水量”的百分比。
* **核心交互区 (水杯容器)：**
    * **视觉呈现：** 屏幕中央展示水杯 UI，内部填充主题色液体。液体表面需实现正弦波浪（Sine Wave）持续起伏的动/态效果，杯底需有微量气泡向上的粒子动画。
    * **滑动手势调节：** * 用户在水杯液面区域上下滑动，可动态调节单次准备记录的饮水量。
        * 容量限制：最小值 `10ml`，最大值 `300ml`。
        * 初始默认容量：`150ml`。
        * 调节步进值（Step）：`10ml`。
    * **触觉反馈 (Haptic Feedback)：** 滑动调节时，每跨越一个档位（10ml）触发一次设备震动。震动强度需与当前选定的水量成正相关（水量越多，震动反馈越强）。
* **底部操作区：**
    * **容量高亮：** 实时显示当前滑动选定的水量数值。
    * **确认按钮：** 采用圆角矩形设计。点击后，立即记录当前水量与当前系统时间，及顶部进度条更新。

### 2.2 历史记录界面 (日历页)
* **整体布局：** 纵向排列的三个卡片式（Card）组件。
* **卡片一：数据总览**
    * 累计记录饮水的天数。
    * 历史记录的饮水总量。
* **卡片二：月度日历**
    * 展示当前月份的日历，左右滑动可以切换月份。
    * **视觉指示器：** 每个日期数字外围包裹一个环形进度条（Circular Progress）。
        * 未记录或 0%：进度条为 0，日期数字显示为灰色。
        * 部分完成 (<100%)：按实际达标百分比显示环形进度。
        * 已达标 (≥100%)：环形完整闭合，且日期数字与圆环均高亮显示为“主题色液体颜色”。
* **卡片三：单日明细与编辑**
    * **联动展示：** 选中日历中某一天后，此卡片刷新展示该日的全部饮水记录明细（时间 + 进水量）。
    * **滚动加载：** 列表超出屏幕高度时，支持向上滑动浏览。
    * **数据管理：** 允许用户直接在此区域修改特定记录的具体时间与进水量。修改保存后，卡片一、二的数据及UI状态需实时同步刷新。

### 2.3 设置界面
* **目标设定：** 配置每日进水总目标（如 2000ml）。
* **主题与个性化：**
    * **颜色自定义：** 支持输入 HEX 色值代码（如 `#00BCD4`），全局更新 App 的液体主题色。
    * **外观模式：** 支持切换深色模式（纯黑背景）与浅色模式（纯白背景）。
* **数据管理：**
    * 提供“导出历史记录”与“导入历史记录”功能，文件格式标准为表格文件（`.csv` 或 `.xlsx`）。
* **关于页脚：**
    * 页面最底部居中显示文字 `power by sakana`。

## 3. UI/UX 与动画规范

* **视觉风格：** 优美、简洁（Minimalist）。减少刚性线条，多采用大面积留白与柔和的卡片阴影。
* **交互动画要求 (Jetpack Compose 实现)：**
    * **液态物理特性：** 水位上升/下降、波浪晃动需具备物理弹簧阻尼感（Spring Physics），避免生硬的线性变化。
    * **操作反馈：** 按钮点击、数值跳动需配备轻快的微缩放或数字滚动动画。
    * **页面路由：** 页面之间的切换需包含流畅的平滑过渡效果（如 Shared Element 共享元素动画或优雅的滑动淡入淡出）。
